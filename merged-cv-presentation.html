Supervised Learning: Mathematical Definition

In supervised learning, we have:

  • Input space \(\mathcal{X}\) and output space \(\mathcal{Y}\)
  • Unknown target function \(f: \mathcal{X} \rightarrow \mathcal{Y}\)
  • Training data \(D = \{(x_1, y_1), ..., (x_n, y_n)\}\) where \(y_i = f(x_i)\)
  • Hypothesis space \(\mathcal{H}\) of functions \(h: \mathcal{X} \rightarrow \mathcal{Y}\)

Goal: Find \(h \in \mathcal{H}\) that approximates \(f\) well on unseen data.

Formula Legend

D = {(x‚ÇÅ, y‚ÇÅ), ‚Ä¶, (x_n, y_n)} where:

  • D is your training dataset (like a textbook)

  • x values are questions or examples

  • y values are correct answers

  • n is how many examples you have

Unsupervised Learning: Mathematical Definition

In unsupervised learning, we have:

  • Input space \(\mathcal{X}\)
  • Training data \(D = \{x_1, ..., x_n\}\) (unlabeled)
  • A notion of similarity or distance between points in \(\mathcal{X}\)

Goal: Discover patterns or structures in \(D\) without explicit labels.

Most model as optimization problems:

For our unsupervised learning example:

  • Define an objective function \(L(D; \theta)\) that measures how well parameters \(\theta\) capture the data structure
  • For clustering: minimize within-cluster distances
  • For dimensionality reduction: minimize information loss
  • For density estimation: maximize data likelihood

The issue: These objective functions are often non-convex and require iterative optimization methods like gradient descent to find good solutions.

Formula Legend

L(D; Œ∏) means:

  • D is your data

  • Œ∏ (theta) represents the patterns we‚Äôre trying to find

Gradient Descent: Basics

An iterative optimization algorithm:

  1. Start with initial parameter guesses
  2. Compute the gradient of the loss function
  3. Update parameters in the opposite direction of the gradient
  4. Repeat until convergence

Gradient Descent: Mathematical Foundation

For a function \(f(\theta)\), the gradient is defined as:

\[\nabla f(\theta) = \begin{bmatrix} \frac{\partial f}{\partial \theta_1} \\ \frac{\partial f}{\partial \theta_2} \\ \vdots \\ \frac{\partial f}{\partial \theta_n} \end{bmatrix}\]

Formula Legend

Œ∏_{t+1} = Œ∏_t - Œ±‚àáf(Œ∏_t) means: - Œ∏_t is current position - Œ± (alpha) is step size - ‚àáf is the direction to move

The update rule is:

\[\theta_{t+1} = \theta_t - \alpha \nabla f(\theta_t)\]

Where \(\alpha\) is the learning rate.

Formula Legend

Œ∏_{t+1} = Œ∏_t - Œ±‚àáf(Œ∏_t) means:

  • Œ∏_t is current position

  • Œ± (alpha) is step size

  • ‚àáf is the direction to move

def iou_animation():
    def calculate_iou(box1, box2):
        # Calculate intersection coordinates
        x1 = max(box1[0], box2[0])
        y1 = max(box1[1], box2[1])
        x2 = min(box1[0] + box1[2], box2[0] + box2[2])
        y2 = min(box1[1] + box1[3], box2[1] + box2[3])
        
        # Calculate areas
        intersection = max(0, x2 - x1) * max(0, y2 - y1)
        area1 = box1[2] * box1[3]
        area2 = box2[2] * box2[3]
        union = area1 + area2 - intersection
        
        return intersection / union if union > 0 else 0

    # Create figure and axis
    fig, ax = plt.subplots()
    ax.set_xlim(0, 10)
    ax.set_ylim(0, 10)

    # Ground truth box (fixed)
    gt_box = [2, 2, 4, 4]
    gt_rect = Rectangle((gt_box[0], gt_box[1]), gt_box[2], gt_box[3], 
                       fill=False, color='green', label='Ground Truth')
    ax.add_patch(gt_rect)

    # Predicted box (will move)
    pred_box = [1, 1, 4, 4]
    pred_rect = Rectangle((pred_box[0], pred_box[1]), pred_box[2], pred_box[3], 
                         fill=False, color='red', label='Prediction')
    ax.add_patch(pred_rect)

        pred_box[0] = 1 + frame * 0.1
        pred_box[1] = 1 + frame * 0.1
        pred_rect.set_xy((pred_box[0], pred_box[1]))
        iou = calculate_iou(gt_box, pred_box)
        ax.set_title(f'IoU: {iou:.2f}')
        return pred_rect,

    anim = FuncAnimation(fig, update, frames=50, interval=100, blit=True)
    plt.legend()

def visualize_dice():
    # Create sample ground truth and prediction
    size = 100
    gt = np.zeros((size, size))
    pred = np.zeros((size, size))
    
    # Create circles
    x, y = np.ogrid[:size, :size]
    center = size/2
    radius = size/4
    
    # Ground truth circle
    circle_gt = (x - center)**2 + (y - center)**2 <= radius**2
    gt[circle_gt] = 1
    
    # Predicted circle (slightly offset)
    radius_pred = size/4 - 5
    circle_pred = (x - (center+5))**2 + (y - (center+5))**2 <= radius_pred**2
    pred[circle_pred] = 1
    
    # Calculate DICE
    intersection = np.sum(pred * gt)
    dice = (2. * intersection) / (np.sum(pred) + np.sum(gt))
    
    # Visualize
    plt.figure(figsize=(15, 5))
    plt.subplot(131)
    plt.imshow(gt, cmap='gray')
    plt.title('Ground Truth')
    
    plt.subplot(132)
    plt.imshow(pred, cmap='gray')
    plt.title('Prediction')
    
    plt.subplot(133)
    plt.imshow(gt + pred, cmap='RdYlBu')
    plt.title(f'Overlay (DICE = {dice:.3f})')
    
    plt.tight_layout()
    plt.show()

Vision Losses

Classification Losses

  1. Cross-Entropy Loss: \[L_{CE} = -\sum_{i=1}^K y_i \log(\hat{y}_i)\]

Formula Legend

Cross-Entropy Loss: L_{CE} = -‚àëy_i log(≈∑_i)

  • y_i is the true answer (0 or 1)

  • ≈∑_i is how confident our guess is

  • The sum adds up all our mistakes

  1. Focal Loss: \[L_{FL} = -\sum_{i=1}^K (1-\hat{y}_i)^\gamma y_i \log(\hat{y}_i)\]

Key Takeaway

State Space Models are a newer, faster way to process images that work more like a flowing river than a series of steps - information flows smoothly and efficiently through the system.

  1. Tversky Loss: \[L_{Tversky} = 1 - \frac{|X \cap Y|}{|X \cap Y| + \alpha|X \setminus Y| + \beta|Y \setminus X|}\]

Formula Legend

L_{Tversky} = 1 - |X ‚à© Y| / (|X ‚à© Y| + Œ±|X| + Œ≤|Y|) means:

  • X ‚à© Y is the overlap between prediction and truth

  • Xis where we predicted wrong (false positives)

  • Yis what we missed (false negatives)

  • Œ± and Œ≤ control how much we care about each type of error

Vision Metrics

Classification Metrics

  • Accuracy
  • Precision
  • Recall
  • F1-Score
  • Mean Average Precision (mAP)

Accuracy

‚ÄúOut of all our guesses, how many did we get right?‚Äù

  • When we make 100 predictions, and get 90 correct, accuracy is 90%
  • Best for balanced datasets
  • Formula: (True Positives + True Negatives) / Total Predictions

Precision

‚ÄúWhen we say something is X, how often is it actually X?‚Äù

  • If we flag 10 emails as spam, but only 8 are actually spam ‚Üí 80% precision
  • Focuses on false positives
  • Formula: True Positives / (True Positives + False Positives)

Recall

‚ÄúOut of all the real cases of X, how many did we catch?‚Äù

  • If there are 20 spam emails and we catch 15 ‚Üí 75% recall
  • Focuses on false negatives
  • Formula: True Positives / (True Positives + False Negatives)

F1-Score

‚ÄúA balanced score between precision and recall‚Äù

  • Combines precision and recall into a single number
  • Perfect score = 1.0, worst score = 0.0
  • Formula: 2 * (Precision * Recall) / (Precision + Recall)

Mean Average Precision (mAP)

‚ÄúHow well we do across different confidence levels‚Äù

  • Used heavily in object detection
  • Considers performance at various threshold levels
  • Helps evaluate confidence of predictions

When to Use Each?

  • Accuracy: When classes are balanced and mistakes in either direction are equally bad
  • Precision: When false positives are costly (spam detection)
  • Recall: When false negatives are costly (disease detection)
  • F1-Score: When you need balance between precision and recall
  • mAP: For object detection or ranked retrieval tasks

def improved_patch_visualization():
    # Create a more visible test image (8x8)
    image = np.zeros((8, 8))
    image[2:6, 2:6] = 0.5  # Gray square
    image[3:5, 3:5] = 1.0  # White center
    patch_size = 2
    
    fig, (ax1, ax2, ax3) = plt.subplots(1, 3, figsize=(15, 5))
    
    # Original image
    ax1.imshow(image, cmap='gray', interpolation='nearest')
    ax1.set_title('Original Image')
    ax1.grid(True, which='major', color='red', linewidth=0.5)
    
    # Image with patch grid
    ax2.imshow(image, cmap='gray', interpolation='nearest')
    for i in range(0, 8, patch_size):
        ax2.axhline(y=i-0.5, color='r', linestyle='-', linewidth=2)
        ax2.axvline(x=i-0.5, color='r', linestyle='-', linewidth=2)
    ax2.set_title('Image with 2x2 Patches')
    
    # Extracted patches
    patches = []
    for i in range(0, 8, patch_size):
        for j in range(0, 8, patch_size):
            patch = image[i:i+patch_size, j:j+patch_size]
            patches.append(patch)
    
    # Create a grid of extracted patches
    n_patches = len(patches)
    grid_size = int(np.sqrt(n_patches))
    patch_grid = np.zeros((grid_size * patch_size, grid_size * patch_size))
    
    for idx, patch in enumerate(patches):
        i = (idx // grid_size) * patch_size
        j = (idx % grid_size) * patch_size
        patch_grid[i:i+patch_size, j:j+patch_size] = patch
    
    ax3.imshow(patch_grid, cmap='gray', interpolation='nearest')
    ax3.set_title('Extracted Patches')
    ax3.grid(True, which='major', color='red', linewidth=0.5)
    
    plt.tight_layout()
    plt.show()

Vision Transformers

Vision Transformer architecture and operation

ViT Components Explained

  1. Image Patching:
    • Divides input image into fixed-size patches (typically 16x16)
    • Reduces computational complexity
    • Enables transformer processing

ViT Components Explained

  1. Patch Embedding:
    • Linear projection of flattened patches
    • Transforms patches into token embeddings
    • Preserves spatial information

ViT Components Explained

  1. Positional Encoding:
    • Adds location information to patches
    • Preserves spatial relationships
    • Usually learnable or sinusoidal

ViT Components Explained

  1. Transformer Blocks:
    • Multi-Head Attention (MHA)
    • MLP layers
    • Layer Normalization

Vision Transformer Variants

  • DeiT: Data-efficient training
  • Swin Transformer: Hierarchical feature learning
  • PVT: Pyramid structure
  • ConvNeXT: Hybrid CNN-Transformer
  • Hybrid Architectures: Best of both worlds
